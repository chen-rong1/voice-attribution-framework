# 安全策略

感谢你帮助改进 `voice-attribution-framework` 的安全性。

## 支持范围

当前仓库尚未提供长期维护版本矩阵。默认以主分支的最新公开代码为主要维护对象。

## 报告内容

如果你发现以下问题，建议按安全问题处理：

- 依赖或默认配置导致的明显安全风险
- 本地文件读取、路径处理或模型加载中的风险行为
- 可能导致敏感路径、环境信息或资产暴露的问题
- 供应链相关问题，例如模型资产来源说明缺失或分发边界不清

## 报告方式

请不要在公开 issue 中直接披露未修复的安全细节。

建议通过以下方式联系维护者：

- GitHub 私信或仓库维护者公开联系方式
- 如果仓库后续补充专用安全邮箱，应优先使用该渠道

如果当前没有私密渠道，可先提交一条不包含细节的公开 issue，仅说明存在需要私下沟通的安全问题。

## 报告时建议提供的信息

- 问题类型和影响范围
- 复现步骤
- 受影响的文件、模块或配置
- 可能的利用条件
- 如有可行建议，可附带修复方向

## 处理原则

- 维护者会尽量确认问题真实性和影响范围
- 在修复发布前，建议避免公开披露可直接利用的细节
- 如问题涉及模型资产或第三方依赖，将同步评估其许可证和分发影响
