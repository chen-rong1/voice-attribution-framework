# 贡献指南

感谢你对 `voice-attribution-framework` 的关注。

本项目当前定位为一个面向已知说话人集合的离线声纹归属基座。提交变更前，建议先阅读 `README.md`、`models/README.md` 和相关测试文件，确保变更方向与仓库边界一致。

## 贡献范围

欢迎以下类型的贡献：

- 修复明确的缺陷
- 改进已有测试或补充缺失的高价值测试
- 提升可维护性、可读性和稳定性
- 改进通用配置、CLI 和文档
- 完善模型来源说明和公开分发说明

当前不建议直接向本仓库提交以下内容：

- 一次性的业务专项脚本
- 真实业务数据、实验结果或内部文档
- 与当前仓库边界无关的大范围重构
- 未说明来源和许可证的模型资产

## 提交流程

建议按以下流程提交变更：

1. 先阅读现有 `README.md` 和相关模块代码，确认改动范围
2. 如有必要，先创建 issue 描述问题、背景和目标
3. 使用独立分支开发
4. 为改动补充必要测试，或说明不补测试的原因
5. 提交 PR，并在描述中写清：
   - 改了什么
   - 为什么改
   - 如何验证
   - 是否涉及模型、配置或文档边界调整

## 代码与文档要求

- 保持改动聚焦，避免一次 PR 混入不相关修改
- 优先复用现有抽象和模块边界
- 对外文档使用中性、可公开、可审计的表述
- 不要在公开仓库中引入本地绝对路径、真实业务命名或敏感数据
- 如果涉及模型文件，必须同步更新 `models/MODEL_PROVENANCE.md`

## 测试

提交前建议至少运行：

```bash
pytest
```

如果改动只影响局部模块，也可以补充说明实际验证范围。

## Pull Request 检查项

提交 PR 前，建议确认：

- 改动范围与仓库定位一致
- 代码和文档已同步更新
- 测试已运行，或已说明未运行原因
- 未提交 `docs/`、业务实验脚本、输出产物或缓存文件
- 如涉及模型资产，来源说明和许可说明已同步更新
